前言

1、什么是容器技术？
容器技术是一种类似于传统虚拟机的虚拟化技术，但是更加轻量化。

2、什么是docker？
docker是容器技术中的一种，也用的很多，但是容器并不止这一种，比如还有rkt等。

centos7搭建docker

安装docker

①卸载老版本的docker及其相关依赖
sudo yum remove docker docker-common container-selinux docker-selinux docker-engine
②安装yum-utils，它提供了yum-config-manager，可用来管理yum源
sudo yum install -y yum-utils
③添加yum源
sudo yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo
④更新索引
sudo yum makecache fast
⑤安装docker-ce
sudo yum install docker-ce
⑥启动docker
sudo systemctl start docker
⑦验证是否安装成功
sudo docker --version

安装docker-compose

sudo curl -L “https://github.com/docker/compose/releases/download/1.23.2/docker-compose-$(uname -s)-$(uname -m)” -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compos
docker-compose --version

centos6安装docker

1、首先升级kernel
因为centos没有封装ssl，先装一个ssl。
yum -y install curl nss openssl
centos6.4自带的内核版本是2.6.32-358.23.2e16.x86_64，而docker要求内核版本大于3.0，推荐3.8以上的内核。
uname -a #查看内核版本
rpm --import https://www.elrepo.org/RPM-GPG-KEY-elrepo.org #导入公钥数字证书
rpm -ivh https://www.elrepo.org/elrepo-release-6-9.el6.elrepo.noarch.rpm
yum -y --enablerepo=elrepo-kernel install kernel-lt
升级完成后可以在文件/etc/grub.cof看到内核的配置信息，将default1修改为default0，使系统默认使用下表为0的内核。重启即可使用高版本的内核启动该操作系统。

2、安装docker
wget https://get.docker.com/rpm/1.7.0/centos-6/RPMS/x86_64/docker-engine-1.7.0-1.el6.x86_64.rpm
yum install -y libcgroup.x86_64 glibc.i686 sqlite.i68
rpm -ivh docker-engine-1.7.0-1.el6.x86_64.rpm

3、配置
vi /etc/sysconfig/docker/sysconfig/docker)
other_args="$other_args --registry-mirror=http://556b149e.m.daocloud.io"
other_args="$other_args --log-driver=none"

4、启动
service docker start

5、异常报错
如果出现以下问题
docker: relocation error: docker: symbol dm_task_get_info_with_deferred_remove, version Base not defined in file libdevmapper.so.1.02 with link time reference
解决办法
sudo cd /etc/yum.repos.d sudo wget public-yum.oracle.com/public-yum-ol6.repo sudo wget public-yum.oracle.com/RPM-GPG-KEY-oracle-ol6 -O /etc/pki/rpm-gpg/RPM-GPG-KEY-oracle sudo yum update -y device-mapper-libs sudo yum update -y libcgroup

基础命令

注：
docker命令前面往往要加以“docker”开头。
docker命令需要root权限

1、docker search 镜像名
在docker hub上搜索指定镜像：
2、docker pull 镜像名称
找到需要的镜像后，用该命令进行下载：
3、docker images
查看本地当前存在的镜像
TAG是容器的标签，用来区分不同的容器，如果不填写的话默认是latest，如果本地不存在latest版本则会自行下载。
IMAGE ID是镜像的16位短ID。
4、docker run -itd --name 容器名称 -p 主机端口:容器端口 -p......镜像名称:标签 /bin/bash
这条命令基本能满足环境的搭建了
-p 是用来映射端口的，不需要映射的情况下可以忽略。
这样就成功创建了一个tomcat的容器，并将容器的8080、8009端口映射到主机的8080、8009端口上。
下面这一长串是容器的完整ID。
5、 docker ps
列出当前正在运行的容器
6、docker stop/start 容器名/ID
停止或启动容器，若是填写容器名则要全写上去，如果用的是ID，这里的ID并不需要完整的ID或者16位短ID，只需要能与其他容器区分开即可，就算你只写第一个字符都是可以的，只要能区分开。
7、docker ps -a
列出所有的容器，包括运行中的和停止的。
8、docker exec -it 容器名/ID /bin/bash
进入到容器中，进入后可以看到区域名称变成了容器的短ID。
9、ctrl+d
退出并停止容器
ctrl+p(按完放开)+q
退出而不停止容器
进入容器后，往往容器中会缺少个别常用命令，例如ifconfig、wget等，只需要正常下载即可。若是没有安装ifconfig的话，可以在容器外使用docker inpect 容器名|grep IPAddress查看容器信息，里面有ip。
11、docker cp 主机中的文件目录 容器名:容器中的目录
将文件从主机拷贝到容器中
12、docker containers cp -a ID:/etc/nginx /root
将容器内文件复制到本地

配置检查

该脚本用于对docker应用服务器的安全合规性检查和配置规范的检测。运行环境为python2，使用了os模块。
脚本检测结果有4种：
1、qualified
2、unqualified
3、not found
4、列表
qualified是某点规范的检测结果，定义为合格。
unqualified是某条规范的检测结果，定义为不合格。
not found说明脚本尝试打开配置文件没有找到，出现这种情况的原因可能有以下情况导致的。
①权限问题：有一些配置文件需要一定权限才能到阿凯；
②未能在指定位置找到对应的配置文件；
③确认相应服务是否安装，且安装位置是否有问题（这会导致脚本不能再文档指定的位置找到对应的配置文件）。
列表是某条规范的检测结果，需要个人自己去判断。

需要提升权限的目录（结合实际环境进行相应的配置）：
必须是root用户
docker命令需要使用root权限，需要获得以下几条命令的执行权限：
docker info | grep -e "^storage Driver:\saufs\s$"
