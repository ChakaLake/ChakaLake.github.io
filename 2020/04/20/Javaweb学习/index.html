JSP环境搭建及入门

静态和动态

1、是否随着时间、地点、用户操作的改变而改变。
2、动态网页需要使用到服务端脚本语言（JSP）。

架构

1、CS：Client Server
■如果软件升级，那么全部软件都需要升级
■维护麻烦，需要维护每一台客户端软件
■每一台客户端都需要安装客户端软件
2、BS：Broswer Server
■客户端可以通过浏览器直接访问服务端

tomcat目录详解


■backup：保存了一些配置文件，是在第一次运行了Tomcat服务器以后产生的。它是对服务器进行简单的备份日志。
■bin：主要用来存放Tomcat的命令，很多环境变量也在此处设置。以.sh结尾的代表linux命令；以.bat结尾的代表Windows命令。其中startup代表启动Tomcat命令；shutdown代表关闭Tomcat命令。
■conf：存放tomcat服务器全局配置的各种文件。backup文件夹为它的备份。catalina目录下可以设置默认加载的项目和服务器内存大小；context.xml可以用来配置数据源之类的东西；logging文件记录tomcat服务器运行的日志记录；server.xml可以设置域名、端口号、IP、请求的编码、默认加载的项目；web.xml可以设置tomcat服务器支持的文件类型，默认的初始页面；tomcat-users.xml用来配置管理tomcat的用户与权限。
■lib：存放tomcat服务器所需要的所有jar包。我们在连接数据库时常常烦恼每创建一个项目就要引入一个对应的oracle或mysql的jar包，但是如果你把这个jar包放在此目录下，就可以引入一次，下次再创建项目就不需要再次引入jar包。
■log：用来存放tomcat执行时的日志文件。清楚此文件不会对tomcat服务器产生影响。开始时此文件目录为空，只有运行了tomcat才会在此目录下生成日志文件。
■temp：用来存放tomcat执行过程中产生的临时文件。清除此文件不会影响tomcat服务器。|
■webapps：tomcat默认部署路径。用来存放应用程序，当tomcat启动时会加载webapps目录下的应用程序。当然可以以war包、jar包、 普通文件三种形式发布应用。你也可以把应用程序放在电脑磁盘的任意一个位置，但是需要在conf/catalina/localhost目录下配置文件中进行映射。
■work：用来存放tomcat运行时编译后的项目文件。清空work目录，重启tomcat，可以达到清除tomcat服务器缓存的目的。
■BUILDING.txt、CONTRIBUTING.md、LICENSE、NOTICE、RELEASE-NOTES、RUNNING.txt是tomcat服务器的一些签证，运行，注意的信息。

常见状态码

200：请求成功，一般用于get或post请求
301：资源（网页等）被永久转移到其他URL
404；请求的资源（网页等）不存在
500：内部服务器错误

配置虚拟路径

1、方法一：修改server.xml文件

在server.xml中找到Host标签，在里面添加
<Context path="test1" docBase="D:\JspProject\test1" debug="0" reloadable="true" crossContext="true" />*
■path：虚拟路径（可以给定绝对路径，也可以给定相对于webapps的相对路径），指定访问该web应用的url入口
■docBase：实际路径（可以给定绝对路径，也可以给定相对于webapps的相对路径），指定web应用的文件路径，，如果web采用开放式目录结构，则指定web应用的根目录，如果web应用是个war文件，则指定war文件的路径。
■reloadable：当reloadable=true时，相关文件改变，tomcat先停止web app并释放内存，然后重新加载web app。这样可以省去手工部署web app工程的时间，和开发工具一起使用可以提高工作效率
■crossContext：如果想在应用内调用 ServletContext.getContext()来返回在该虚拟主机上运行的其他 web application 的 request dispatcher,设为 true。在安全性很重要的环境中设为 false，使得 getContext()总是返回 null。缺省值为 false。
■ Debug：与这个 Engine 关联的 Logger 记录的调试信息的详细程度。数字越大，输出越详细。如果没有指定，缺省为 0。

2、方法二：添加配置文件

在tomcat根目录下的/conf/catalina/localhost/的路径下新建一个“x.xml”
<Context docBase="D:\JspProject\test1" />
■docBase指向应用程序目录，各参数参见方法一中的标签参数（主次此文件名x将作为Context中的path属性值，不管文件里的path属性值如何设置也是无效的。）
■此方法不需要重启tomcat。

Eclipse与web开发

Eclipse集成tomcat

利用Ecliplse创建java web项目

1、在Eclipse左侧空白处，右键选择New–>Dynamic Web Project（或者左上角File–>New–>…）

2、若New的选择列表中没有Dynamic Web Project，则选择Others，然后输入dynamic（一般输入dy就可以），选择Dynamic Web Project


3、输入项目名称，选择运行时环境（中间件环境），选择web相应的版本

4、点击Finish

5、此时如果发现java web工程缺少web.xml文件，那么右键项目名称–>Java EE Tools–>Generate Deployment descriptor stub

6、最后项目成功创建

注意：
浏览器可以直接访问web Content中的文件，但WEB-INF中的文件无法通过客户端（浏览器）直接访问，只能通过请求转发来访问。并不是任何内部跳转都能访问WEB-INF，原因是跳转有两种方式：请求转发、重定向。

在servers面板新建一个tomcat实例，再在该实例中部署项目（右键–>add）之后运行。一般建议将eclipse中的tomcat与本地tomcat的配置信息保持一致，将eclipse中的tomcat上设置为托管模式（第一次创建tomcat实例之后，双击，）。

JSP内置对象

OUT

输出对象，用于在web浏览器内输出信息，并且管理应用服务器上的输出缓冲区。在使用out对象输出数据时，可以对数据缓冲区进行操作，及时清除缓冲区的残余数据，为其他的输出让出缓冲区。带数据输出完毕后，要及时关闭数据流。

request

请求对象，代表了客户端的请求信息，主要用于接收通过http协议传送到服务器的数据（包括头信息、系统信息、请求方式以及请求参数等）。request对象的作用域为一次请求。
